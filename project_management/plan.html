MANAKISH BAKERY AI AGENT PLATFORM

Development of a Domain-Specific Conversational AI & Autonomous Fulfillment Infrastructure

1. Executive Summary & System Scope Executive Briefing

Senior Principal Lead Assessment

The Manakish Bakery AI Assistant platform is entering Phase 3 validation. This enterprise project replaces legacy static web ordering with an autonomous, natural-language conversational agent powered by Google Gemini, Node.js/Express, and React 18. The system integrates structured schema tools, dynamic delivery zone boundary calculations, multi-turn working memory state, and durable file-backed persistence mapped to containerized volumes.

The primary project objective is delivering an enterprise-grade agentic ordering engine capable of executing end-to-end customer workflows—menu inquiry, customized order creation, address verification with delivery radius checks, and administrative order state transitions—without human intervention while guaranteeing zero hallucinations on pricing and inventory.

1.1 Core Architectural Modules

  • Conversational Intent Engine: Powered by Gemini 1.5/2.0 via Google Gen AI SDK with strict system prompt boundaries, managing short-term session context and multi-turn state.
  • Structured Tool Invocation Layer: Native server-side schema verification (`check_delivery_zone`, `create_bakery_order`, `get_bakery_menu`, `update_order_status`).
  • Durable JSON Persistence Store: High-reliability file storage (`/app/data/orders-db.json`) with auto-healing seed data initialization and atomic write locks.
  • Operational Telemetry & Observability Panel: Real-time trace logging dashboard rendering turn-by-turn agent thought steps, tool inputs/outputs, and token latency breakdowns.

3. Technical Risk Matrix & Pragmatic Mitigation Strategies Risk Analysis

Deploying autonomous LLM agents into transactional production workflows introduces operational failure modes distinct from traditional software architectures. Below is the pragmatic risk register with severity scoring (Scale 1 = Low, 5 = High) and trade-off mitigations:

Identified Technical Risk Prob. Imp. Prio. Prevention Strategy Engineering Contingency & Fallback
LLM Tool Call Hallucination / Invalid Parameter Ingestion 3 5 15 Strict JSON Schema bounds in function definitions; server-side sanitization prior to tool invocation. Trap schema validation errors in Express layer; re-inject structured error message into conversation turn for self-correction.
JSON Database Concurrent File Write Race Conditions 2 5 10 Atomic file write wrapper with memory locks (`fs.writeFileSync` temp swap pattern). In-memory order queuing with async mutex lock before flushing state changes to container volume.
Gemini API Rate Limits / Provider Outage 3 3 9 Implement exponential backoff retry in Gen AI SDK wrapper; token budget monitoring. Graceful degradation to structured rule-based fallback dialog prompting customer for human call assistance.
Container State Loss Across Restarts 2 4 8 Bind `/app/data` directory to explicit Docker named volume (`bakery_data`) in `compose.yaml`. Auto-healing startup sequence that recreates default seed state if volume initialization fails.
Session Context Window Drift / Hallucinated History 3 2 6 Keep system instructions lean; pass current pending order state explicitly in context. Client session reset trigger and mandatory confirmation step before order write execution.
