The Manakish Bakery AI Assistant project consists of developing an enterprise-grade web software system enabling autonomous, conversational online ordering powered by Google Gemini, Express, and React 18.

2. Strategic Justification Business Case

A fixed-price commercial contract of $29,000 has been established with the client sponsor. The baseline estimated internal effort budget is $7,040, yielding a strong projected return on investment.

Beyond direct financial returns, the system represents:

  • Strategic Positioning: Establishing an early leader footprint in autonomous agentic e-commerce.
  • Product Replication: High reusability for turn-key deployment across additional food service & retail verticals.
  • Portfolio Expansion: Enhancing SoftSolution's portfolio with production-tested LLM function calling and real-time trace telemetry engines.

3. Project Objectives Targets

Develop and deploy a web-based conversational ordering system that:

  • Allows customers to browse menu categories, prices, and customizations via natural language or UI.
  • Executes autonomous delivery zone boundary validation (<5km Zone 1, 5-10km Zone 2, >10km rejection).
  • Supports structured order submission with unique order tracking IDs (`BAKE-XXXX`).
  • Saves transactional state to a persistent, atomic JSON database with container volume mapping.
  • Provides an operational telemetry trace inspector and administrative order state dashboard.

4. Expected Deliverables Outputs

  • Executable Web Application System (Version 1.0 - React + Express + Gemini SDK).
  • Installed containerized deployment package with Dockerfile and named persistence volume setup.
  • Live operational trace logging panel for turn-by-turn agent inspection.
  • Complete technical architecture documentation, tool schema definitions, and REST APIs.
  • Test & validation report verifying zero price hallucinations and boundary edge cases.

5. Scope Definition Boundaries

Included Scope

  • Conversational agent & memory state loop.
  • Typed tool schema execution (`server.ts`).
  • Spatial Haversine delivery zone validation.
  • Atomic file database (`orders-db.json`).
  • Admin fulfillment dashboard & trace inspector.
  • Docker container packaging & health probes.

Excluded Scope

  • Native mobile iOS/Android apps.
  • Physical point-of-sale (POS) hardware syncing.
  • Third-party payment gateway merchant setup.
  • External SMS/Email marketing automation tools.
  • GPS driver vehicle fleet hardware tracking.

6. Assumptions and Restrictions Constraints

  • SoftSolution Cloud Run container infrastructure is fully accessible.
  • Google Gemini API keys (`GEMINI_API_KEY`) provided and authenticated.

Restrictions:
